# ADR：为演示战斗计算引入 SoonFx

- 状态：已接受（仅演示范围）
- 日期：2026-03-09

## 背景

我们需要在一天内交付一个可复现的战斗数值计算链路，并具备自动化初始化、测试、本地部署和回滚能力（Windows 环境）。

## 决策

- 采用 Node.js + TypeScript 作为运行时宿主。
- 通过适配层集成 `@soonfx/engine`；当运行时 API 不可用时走确定性后备计算路径。
- 暴露最小 HTTP API：`GET /health`、`POST /calc-demo`。
- 提供 PowerShell 自动化脚本：bootstrap/deploy/rollback。

## 备选方案

1. 纯 Python 实现  
优点：团队生态熟悉。  
缺点：与 SoonFx 对齐较弱，集成复杂度更高。

2. 直接耦合完整游戏引擎  
优点：更接近最终生产形态。  
缺点：无法在 1 天内安全完成。

## 风险

- SoonFx 不同版本之间运行时 API 可能存在差异。
- Windows 服务管理需要管理员权限。
- 仅本地部署路径缺少生产级可观测性。

## 缓解措施

- 保持引擎适配边界和 fallback 模式。
- 增加 20 组 golden 回归用例。
- 补充明确的运行手册和生产迁移检查清单。
