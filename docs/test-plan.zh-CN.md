# 测试计划

## 目标

验证战斗计算结果的确定性、输入处理正确性，以及本地部署可用性。

## 范围

- 公式计算的单元测试与回归测试。
- `/health` 与 `/calc-demo` 的 API 合约测试。
- `/calc-demo` 的本地性能基线测试。
- 各执行一次部署验证与回滚验证。

## 测试项

1. 功能测试  
固定输入必须返回固定结果。  
非法输入必须返回 HTTP 400 及错误详情。

2. 回归测试  
20 组本地 golden case，要求与期望值完全一致。

3. 性能测试  
执行 `npm run test:perf`（500 请求，并发 25）。  
记录 p50、p95、错误数。

4. 部署测试  
新装部署成功。  
覆盖部署成功。  
服务重启成功。  
回滚成功且健康检查通过。

## 通过标准

- `npm run test` 全部通过，无失败用例。
- 性能测试 `errorCount = 0`。
- 部署后与回滚后，健康检查均返回 `status = ok`。

## 故障注入

- 向 `/calc-demo` 发送非法 JSON 字段类型。
- 删除 `node_modules` 后重跑 bootstrap，模拟依赖缺失场景。
